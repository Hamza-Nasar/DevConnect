We take the security of our web application seriously. If you discover a security vulnerability, we appreciate your help in responsibly disclosing it.

### What to Include

To help us investigate efficiently, please include:

* A clear description of the vulnerability
* Steps to reproduce the issue (proof of concept if possible)
* Affected versions and components
* Any potential impact you have identified

### Response Timeline

* **Acknowledgement:** Within 48 hours of receiving your report
* **Initial assessment:** Within 3‚Äì5 business days
* **Status updates:** Provided as the investigation progresses

### Resolution Process

* If the vulnerability is **accepted**, we will work on a fix and coordinate a responsible disclosure timeline.
* If the report is **declined** (e.g., not a security issue or already fixed), we will explain the reason clearly.
* Once resolved, a security update or patch will be released for supported versions.

### Responsible Disclosure

We kindly request that you:

* Give us reasonable time to investigate and fix the issue before public disclosure
* Avoid exploiting the vulnerability beyond what is necessary for testing
* Act in good faith to protect users and their data

### Scope

This security policy applies specifically to **DevConnect**, a social networking application where users can create profiles, connect with others, share content, and interact in real time.

The scope includes:

* User authentication and authorization
* Profiles, posts, comments, likes, and connections
* Messaging and notifications
* APIs, backend services, and databases
* Web and mobile client applications

---

### Out of Scope

The following are generally considered out of scope:

* Issues requiring physical access to user devices
* Social engineering attacks (phishing, spam, impersonation)
* Denial-of-service attacks without demonstrable impact
* Vulnerabilities in third-party services not under our control

---

### Data Protection & Privacy

DevConnect is committed to protecting user data. Any vulnerability that could lead to:

* Unauthorized access to personal data
* Account takeover
* Data leakage or privilege escalation

will be treated as **high priority**.
